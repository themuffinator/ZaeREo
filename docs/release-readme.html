This checked-in file is the development template. A release candidate must render a version- and mode-specific README that removes inapplicable modes and commands.

Importer-kit mode

When the independent code-rights gate permits it, this archive contains the game module and importer, but no Zaero maps or media. It is deliberately not playable until completed from your own legitimate Zaero installation.

cd zaereo
pwsh -File .\tools\complete_importer_kit.ps1 `
  -ZaeroLegacyRoot "D:\Games\Zaero" `
  -EngineRoot "D:\Games\Quake II Rerelease"

The importer verifies the three known retail PAK SHA-256 hashes, downloads nothing, builds pak1.pak locally, retains the nine required loose sprite/video files, and merges the Zaero map database fragment against your supported Rerelease data without redistributing its base database. Generated outputs are recorded in a separate import-ownership manifest.

Local-full mode

This mode is for private developer testing. It is produced only after the same retail-hash verification and contains locally imported commercial media.

Do not redistribute a local-full archive—ever. This mode is permanently private and cannot be promoted or renamed into a public release, even if rights later change.

The current prototype packager still combines imported content into pak0.pak; it does not yet satisfy the target pak0/pak1 ownership split described below and is development evidence only.

Asset-full mode

This distinct public mode does not exist unless both code and media rights are affirmatively cleared. It must be built from reviewed distributable inputs, never from a user's local-full import.

Install and launch

  1. Verify the downloaded archive against its accompanying SHA-256 checksum file.
  2. Install the top-level zaereo/ directory under %USERPROFILE%\Saved Games\Nightdive Studios\Quake II\. Never extract it over baseq2.
  3. If this is an importer kit, run the completion command above.
  4. Launch through the mod selector when supported, or with +set game zaereo +exec zaerostart.cfg.

A portable install beside the game is an explicit advanced override, not the default. The roadmap requires a packaged tools/manage_install.ps1 to preview/install/repair/update/uninstall only manifest-owned bytes while preserving saves, local configuration, and import-owned output. That player manager is not implemented in the current development tree, so this template makes no managed-update claim until a rendered package actually contains and tests it. Never delete the whole directory as an update shortcut.

Importer kits also contain IMPORT_ASSETS.txt and tools/. Completed importer kits and private local-full builds keep imported effective media in local pak1.pak plus the nine verified loose files under .zaereo-install/import-ownership.json; they never fold those bytes into package-owned pak0.pak. Asset-full media, if ever cleared, is package-owned instead.

Support evidence

When reporting a defect, include the exact version and commit, map, skill, single-player/co-op/deathmatch mode, reproduction steps, save if safe, and logs. Do not attach original Zaero media.
