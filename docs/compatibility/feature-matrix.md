# Feature compatibility matrix

- Baseline: supplied Zaero source/content versus legacy Quake II
- Target: Quake II Rerelease game API 2023, Windows x64 first
- Matrix status: audited backlog and execution ledger
- Gameplay implementation status: **in progress; see per-row evidence**

This matrix tracks observable behavior, not file-copy progress. PARITY reproduces
legacy behavior, ADAPT preserves intent through a Rerelease-native design, and
FIX corrects a demonstrated defect while preserving valid behavior with a test.

Statuses are NOT STARTED, IN PROGRESS, IMPLEMENTED, VERIFIED, or DEFERRED:

- **IN PROGRESS** means at least one clause, integration, or required proof in
  the row is incomplete;
- **IMPLEMENTED** means the complete row code exists, compiles, and its focused
  automated contracts pass, but the required live map/mode/save proof has not
  yet closed the row;
- **VERIFIED** means the named proof has passed on the intended maps, modes,
  timing, multiplayer, and save dimensions; and
- **DEFERRED** requires an approved decision that keeps the 1.0 parity outcome
  coherent.

The planning audit proves scope only. A source link or static test alone is not
runtime verification.

## Update rule

A pull request may move a row to:

- IMPLEMENTED only when the complete row code exists, compiles, its focused
  contracts pass, and only the required live proof remains;
- VERIFIED only when the named proof passes against the intended maps/modes and
  its evidence is linked; or
- DEFERRED only when an approved decision explicitly removes it from the current
  release scope without contradicting the 1.0 parity outcome.

When one implementation covers several rows, each row still links the exact
test that proves its own observable contract.
